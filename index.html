Software Design & Engineering

The artifact selected for enhancement is the CS 465 Full Stack project Travlr Express, a complete travel booking and administration platform built on the MEAN stack (MongoDB, Express, Angular, Node.js). The original application provided core CRUD operations, server-rendered public pages, and an Angular-based administration interface. It was chosen for enhancement because it spanned full-stack concerns, backend architecture, API design, authentication, frontend integration, and deployment, presenting numerous opportunities to apply professional software engineering practices.

The enhancement focused on strengthening the applicationâ€™s security posture, maintainability, and documentation. Key technical changes included implementing a robust JWT-based authentication scheme using both access and refresh tokens; secure password storage using bcrypt; and role-based access control (RBAC) to separate administrative and regular user privileges. Input validation and sanitization were standardized across routes, and consistent error handling patterns were introduced. To reduce attack surface and improve operability, CORS policies were restricted to trusted origins and rate limiting was applied to critical endpoints.

Architecturally, the codebase was refactored to follow an MVC pattern complemented by a service layer. This separation of concerns reduced duplicated logic, simplified unit testing, and clarified responsibilities between routing, business logic, and data persistence. Middleware was used for cross-cutting concerns (authentication, logging, validation), enabling DRY principles and easier maintenance. An OpenAPI specification was created to describe the API surface and facilitate automated documentation and client generation. JSDoc-style inline documentation and an updated README were added to support onboarding and handoff.

From a security and operations perspective, the project adopted least-privilege database accounts, centralized logging, and clear configuration management to reduce the risks of leaked credentials. These changes were validated through manual testing and review, with considerations for backward compatibility during incremental rollout. The process highlighted the importance of balancing security controls with usability and deployment constraints.

Overall, this enhancement demonstrates applied competencies in secure full-stack engineering: designing authentication and authorization workflows, applying validation and sanitization at multiple layers, refactoring for maintainability, and delivering production-oriented documentation to support stakeholders and future development.

Algorithms & Data Structures

The selected artifact is a CS 330 Computer Graphics and Visualization project: an interactive 3D renderer implemented in C++ and OpenGL. Computer graphics requires rigorous application of algorithms, spatial data structures, and real-time performance optimizations. The original program used brute-force rendering methods that transformed and tested every object each frame, creating a clear opportunity for algorithmic improvement and structural optimization.

The enhancement introduced an Octree-based spatial partitioning system to divide the 3D world into recursively smaller regions, enabling efficient spatial queries and significantly reducing per-frame visibility tests. This Octree is integrated into the SceneManager and combined with frustum culling so only objects potentially visible to the camera are processed for rendering, lowering draw calls and CPU/GPU workload.

Additional algorithmic improvements included distance-based Level of Detail (LOD) to use simplified meshes for distant objects, and caching of transformation computations (for example, rotation matrices) to prevent redundant recalculations within the scene graph traversal. A hierarchical scene graph was introduced using parent-child node relationships to support cohesive transformations and cleaner scene organization.

To validate and quantify the benefit of these enhancements, a performance profiler was implemented. The profiler logs frame rate, visible object counts, draw calls, and section-level timing. These metrics were collected to demonstrate real improvements resulting from the Octree, LOD, and caching optimizations and to inform further tuning decisions.

Challenges included maintaining accuracy during Octree inserts/removals, ensuring consistent transformation propagation through the scene graph, and integrating LOD without visible popping artifacts. These issues were addressed through iterative testing and refinements. The work demonstrates a methodical application of algorithmic principles, thoughtful trade-offs between visual fidelity and performance, and effective communication of technical decisions to diverse audiences.

Databases

The artifact enhanced is the database layer of the Travlr Express project. The original database design supported basic CRUD operations but lacked optimization, robust validation, and advanced data-management features. The database was chosen for enhancement because improvements here would materially increase reliability, scalability, and analytical capability across the application.

Enhancements focused on schema redesign, query optimization, data integrity, aggregation capabilities, and database-level security. Schemas were normalized where appropriate, numeric-string fields were converted to proper numeric types, and a schemaVersion field was introduced to support future migrations and backward compatibility. Relationships between collections (Trips, Rooms, Meals, News, Users) were formalized using references, improving consistency and enabling richer analytical queries.

To address performance, targeted single-field and compound indexes were created based on common query patterns and profiling data. Full-text search support was strengthened for user-facing search features. Cursor-based pagination replaced offset-based techniques to provide efficient and reliable paging for large result sets.

Data validation was significantly improved using comprehensive Mongoose schema rules, custom business validators, and lifecycle hooks to normalize and enforce consistency. Additionally, MongoDB-level JSON schema validators were applied (via collMod) to prevent invalid documents from entering the database outside of application constraints.

Advanced data operations were implemented through aggregation pipelines for reporting and analytics (for example, trip distributions by resort and month, room availability trends, and pricing summaries). These pipelines were surfaced as new API endpoints and integrated into the Angular admin UI. Reusable pagination and full-text search helpers were added to the service layer to keep the code modular and maintainable.

Security improvements included input sanitization, enforcement of least-privilege database user roles, and a Mongoose audit-logging plugin to capture create/update/delete events for accountability. The enhancement process underscored the importance of designing indexes around actual query patterns, planning migrations carefully to maintain backward compatibility, and integrating validation at both application and database levels.
